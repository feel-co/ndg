ndg-nixdoc.

This crate walks one or more .nix source files, locates attribute bindings preceded by /** ... */ doc comments, and returns structured documentation entries. The nixdoc crate is always used to parse doc-comment content into ParsedDoc.

If you want to make Nix doc-comment support optional in a consumer crate, gate the dependency on ndg-nixdoc itself behind a feature flag there.

use ndg_nixdoc::extract_from_file;

let entries = extract_from_file("lib/strings.nix").unwrap();
for entry in &entries {
  println!("{}: {}", entry.attr_path.join("."), entry.raw_comment);
}
