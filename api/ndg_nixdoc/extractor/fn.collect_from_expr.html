Recursively walk an expression node, collecting doc-commented bindings.

Only NODE_ATTR_SET and NODE_LET_IN nodes can directly contain attribute entries. For any other node we descend into all children looking for nested attrsets (e.g. the body of a lambda or a conditional).

path_prefix holds the attribute path segments accumulated from ancestor attrsets so that nested entries are reported with their full dotted path.
