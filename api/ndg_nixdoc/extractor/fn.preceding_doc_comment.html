Function preceding_doc_comment 

Source 
fn preceding_doc_comment(node: &SyntaxNode) -> Option<String>
Expand description

Walk the token stream backwards from node to find the immediately preceding /** ... */ doc comment, skipping whitespace tokens.

Only a single Nixdoc block comment is returned. If the nearest non- whitespace token is a plain # ... line comment or a regular /* */ block comment, None is returned.
