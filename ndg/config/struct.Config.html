Expand description

Configuration for the NDG documentation generator.

Config holds all configuration options for controlling documentation generation, including input/output directories, template customization, search, syntax highlighting, and more. Fields are typically loaded from a TOML or JSON config file, but can also be set via CLI arguments.

§input_dir: Option<PathBuf>

Input directory containing markdown files.

§output_dir: PathBuf

Output directory for generated documentation.

Path to custom template file.

Paths to custom stylesheets.

§script_paths: Vec<PathBuf>

Paths to custom JavaScript files.

§assets_dir: Option<PathBuf>

Directory containing additional assets.

§manpage_urls_path: Option<PathBuf>

Path to manpage URL mappings JSON file.

§title: String

Title for the documentation.

§jobs: Option<usize>

Number of threads to use for parallel processing.

§generate_anchors: bool

Whether to generate anchors for headings.

§generate_search: bool

Whether to generate a search index.

§footer_text: String

Text to be inserted in the footer.

§options_toc_depth: usize

Depth of parent categories in options TOC.

§highlight_code: bool

Whether to enable syntax highlighting for code blocks.

§revision: String

GitHub revision for linking to source files.

§opengraph: Option<HashMap<String, String>>

OpenGraph tags to inject into the HTML head (e.g., {“og:title”: “…”, “og:image”: “…”})

§excluded_files: HashSet<PathBuf>

Files to exclude from sidebar navigation (included files).

§meta_tags: Option<HashMap<String, String>>

Additional meta tags to inject into the HTML head (e.g., {“description”: “…”, “keywords”: “…”})
